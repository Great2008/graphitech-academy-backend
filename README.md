# GraphiTech Academy — Backend

FastAPI backend for GraphiTech Academy.

## Stack
- FastAPI + SQLAlchemy 2.0 + Alembic
- PostgreSQL (via Supabase)
- Groq / Anthropic for AI tutor + course drafting
- Paystack for payments (card, bank transfer, OPay, Palmpay channels)
- Judge0 for the coding playground
- WeasyPrint + qrcode for certificate generation

## Deployment (Render)

Deployed via Docker (needed for WeasyPrint's system dependencies — plain
Python buildpacks can't install Pango/Cairo). No local build required —
Render builds the image on their servers from what you push to GitHub.

**Steps (all done from the Render dashboard, nothing to run locally):**

1. Push this repo to GitHub (`graphitech-academy-backend`)
2. On [render.com](https://render.com) → New → Blueprint → connect the repo.
   Render reads `render.yaml` and sets up the service automatically.
3. Fill in the env vars marked `sync: false` in the Render dashboard:
   - `DATABASE_URL` — your Supabase Postgres connection string, with the
     driver prefix changed to `postgresql+psycopg://` (Supabase gives you
     `postgresql://` — just add `+psycopg` after `postgresql`)
   - `GROQ_API_KEY`, `PAYSTACK_SECRET_KEY`, `PAYSTACK_PUBLIC_KEY`
   - `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`
   - `JUDGE0_API_URL`, `JUDGE0_API_KEY` (once you've set up Judge0)
   - `FRONTEND_URL` — your Vercel frontend URL
   - `JWT_SECRET_KEY` is auto-generated by Render, no action needed
4. Deploy. On every deploy, `entrypoint.sh` runs `alembic upgrade head`
   automatically before starting the server — migrations never need to be
   run manually.
5. Check `https://your-service.onrender.com/health` — should return
   `{"status": "ok", "environment": "production"}`
6. Full interactive API docs at `https://your-service.onrender.com/docs`

If a deploy fails, check the Render logs first — a failed migration will
show up clearly there rather than as a silent broken deploy.

## Local setup (optional — only if you have a computer with Node/Python)

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env   # fill in real values

alembic upgrade head    # run migrations
uvicorn app.main:app --reload
```

API docs available at `http://localhost:8000/docs` once running.

## Project structure

```
app/
  core/       # settings, database session
  models/     # SQLAlchemy models (one file per domain module)
  schemas/    # Pydantic request/response schemas
  routers/    # API routes (added incrementally)
  services/   # business logic (added incrementally)
alembic/      # migrations
```

## Creating a migration

```bash
alembic revision --autogenerate -m "describe the change"
alembic upgrade head
```

## Deployment
Deployed to Render. Set all `.env.example` variables as environment variables
in the Render dashboard — do not commit `.env`.
